49  public interface IMVTNCSFirst {
50  
51  	/**
52  	 * The TNC Server calls this method to indicate that an Integrity 
53  	 * Check Handshake is beginning and solicits messages from IMVs 
54  	 * for the first batch. The IMV SHOULD send any IMC-IMV messages 
55  	 * it wants to send as soon as possible after this method is 
56  	 * called and then return from this method to indicate that it is 
57  	 * finished sending messages for this batch.
58  	 * <p>
59  	 * IMVs are not required to implement this method but any IMV that 
60  	 * implements the IMVTNCSFirst interface MUST implement this 
61  	 * method. An IMV should implement this method if it supports 
62  	 * having the TNCS send the first batch of IMC-IMV messages in the
63  	 * TNC handshake. This is useful when an IMV must send instructions
64  	 * to the IMC before the IMC can send useful integrity messages. 
65  	 * Simple IMVs that do not wish to send an initial message to the 
66  	 * IMC need not implement this method.
67  	 * <p>
68  	 * Since this method was not included in IF-IMV 1.2, many IMVs do 
69  	 * not implement it. TNCSs MUST work properly if an IMV does not 
70  	 * implement this method. Likewise, many TNCSs do not implement 
71  	 * this method so IMVs MUST work properly if a TNCS does not 
72  	 * implement this method. If an IMV implements this method, the 
73  	 * TNCS SHOULD call this method if the TNCS is preparing to send 
74  	 * the first batch of messages in a TNC handshake. Even if the 
75  	 * TNCS and IMV both support this method and have used it for 
76  	 * previous handshakes, a particular handshake may not involve 
77  	 * having the TNCS send the first batch of messages. In that case,
78  	 * this method will not be called for that handshake. The TNCS and
79  	 * IMV MUST properly handle the situation where this method is 
80  	 * supported by both the TNCS and IMV but is not called for a 
81  	 * particular handshake because the TNCS will not be sending the 
82  	 * first batch of messages.
83  	 * <p>
84  	 * As with all IMV methods, the IMV SHOULD NOT wait a long time 
85  	 * before returning from beginHandshake. To do otherwise would 
86  	 * risk delaying the handshake indefinitely. A long delay might 
87  	 * frustrate users or exceed network timeouts (PDP, PEP or 
88  	 * otherwise).
89  	 * <p>
90  	 * @param c the IMVConnection object
91  	 *
92  	 * @throws TNCException if a TNC error occurs
93  	 */
94  	void beginHandshake(IMVConnection c) throws TNCException;
95  }
