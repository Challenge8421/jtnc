48      /**
49       * A call to this method is used to inform a TNCC about the 
50       * set of message types the IMC is able to receive. Often, 
51       * the IMC will call this method from the IMC's initialize method. 
52       * A list of message types is contained in the 
53       * supportedTypes parameter. The supportedTypes parameter 
54       * may be null to represent no message types. 
55       * <p>
56       * All TNC Clients MUST implement this method. 
57       * The TNC Client MUST NOT ever modify the list of 
58       * message types and MUST NOT access this list after 
59       * TNCC reportMessageTypes() method has returned. Generally, 
60       * the TNC Client will copy the contents of this list 
61       * before returning from this method. TNC Clients 
62       * MUST support any message type.
63       * <p>
64       * Note that although all TNC Clients must implement 
65       * this method, some IMCs may never call it if they 
66       * don't support receiving any message types. This is 
67       * acceptable. In such a case, the TNC Client MUST NOT 
68       * deliver any messages to the IMC.
69       * <p>
70       * If an IMC requests a message type whose vendor ID is 
71  	 * TNC_VENDORID_ANY and whose subtype is TNC_SUBTYPE_ANY 
72  	 * it will receive all messages with any message type, 
73  	 * except for messages marked for exclusive delivery to 
74  	 * another IMC. If an IMC requests a message type whose 
75  	 * vendor ID is not TNC_VENDORID_ANY and whose subtype is
76  	 * TNC_SUBTYPE_ANY, it will receive all messages with the 
77  	 * specified vendor ID and any subtype, except for messages 
78  	 * marked for exclusive delivery to another IMC. If an IMC 
79  	 * calls the TNCC's reportMessageTypes or reportMessageTypesLong 
80  	 * method more than once, the message type list supplied in 
81  	 * the latest call supplants the message type lists supplied 
82  	 * in earlier calls.
83       * <p>
84       * @param imc an IMC reporting it message types
85       * @param supportedTypes the messages types an IMC can support
86       *
87       * @exception TNCException if a TNC error occurs
88       */
89  
90  	public void reportMessageTypes(IMC imc, long [] supportedTypes) throws TNCException;
91   
92      /**
93       * IMCs can call this method to ask a TNCC to 
94       * retry an Integrity Check Handshake 
95       * for all current network connections.  
96       * The IMC MUST pass itself as 
97       * the imc parameter and one of the handshake retry reasons
98       * in IMCConnection as the reason parameter. 
99       * <p>
100      * TNCCs MAY check the parameters to make sure they are 
101      * valid and return an error if not, but TNCCs are not 
102      * required to make these checks. The reason parameter 
103      * explains why the IMC is requesting a handshake retry. 
104      * The TNCC MAY use this in deciding whether to attempt 
105      * the handshake retry. TNCCs are not required to honor 
106      * IMC requests for handshake retry (especially since 
107      * handshake retry may not be possible or may interrupt 
108      * network connectivity). An IMC MAY call this method at 
109      * any time, even if an Integrity Check Handshake is 
110      * currently underway. This is useful if the IMC suddenly 
111      * gets important information but has already finished its 
112      * dialog with the IMV, for instance. As always, the TNCC 
113      * is not required to honor the request for handshake 
114      * retry.
115      * <p>
116      * If the TNCC cannot attempt the handshake retry, 
117      * it SHOULD throw a TNCException with result code
118      * TNC_RESULT_CANT_RETRY. If 
119      * the TNCC could attempt to retry the handshake but 
120      * chooses not to, it SHOULD throw a TNCException with result code
121      * TNC_RESULT_WONT_RETRY. The IMC MAY use this information 
122      * in displaying diagnostic and progress messages.
123      *  <p>
124      * @param imc an IMC requesting a retry handshake
125      * @param reason the reason for the handshake request
126      *
127      * @exception TNCException if a TNC error occurs
128      */
129 
130 	public void requestHandshakeRetry(IMC imc, long reason) throws TNCException;
131 
132 }
